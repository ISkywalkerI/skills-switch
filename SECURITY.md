# Security Policy

## Supported Versions

At this stage, security fixes are expected to land on the latest version in the default branch.

## Reporting a Vulnerability

Please do not open a public GitHub issue for security vulnerabilities.

Instead:

1. Contact the maintainer privately
2. Include a clear description of the issue
3. Include reproduction steps, affected files, and impact
4. Include proof-of-concept material only when necessary

If you do not yet have a dedicated private contact channel, add one before public release, such as:

- a security email address
- a GitHub Security Advisory intake flow

## What to Include

- affected version or commit
- environment details
- steps to reproduce
- expected impact
- possible mitigation if known

## Response Expectations

The project should acknowledge valid reports, investigate them, and coordinate a fix before public disclosure when possible.
